# Ethereum Guestbook

Простое учебное веб-приложение, использующее смарт-контракт Ethereum. Пользователь подключает MetaMask, отправляет сообщение в смарт-контракт и может прикрепить небольшую сумму ETH. Веб-интерфейс читает все сообщения и баланс контракта.

## Структура проекта

```text
ethereum_guestbook_project/
├── contracts/
│   └── EthereumGuestbook.sol
├── web/
│   ├── index.html
│   ├── style.css
│   └── app.js
└── README.md
```

## Запуск

1. Откройте Remix IDE.
2. Создайте файл `EthereumGuestbook.sol` и вставьте код из папки `contracts`.
3. Скомпилируйте контракт компилятором Solidity 0.8.20 или выше ветки 0.8.x.
4. Разверните контракт через `Injected Provider - MetaMask` в тестовой сети или через `Remix VM`.
5. Скопируйте адрес развернутого контракта.
6. Откройте `web/index.html` в браузере.
7. Подключите MetaMask, вставьте адрес контракта и нажмите «Сохранить».
8. Отправьте сообщение через форму веб-приложения.

## Основные функции смарт-контракта

- `writeMessage(string text)` — записывает сообщение и принимает ETH через `payable`.
- `getAllMessages()` — возвращает массив всех сообщений.
- `getMessagesCount()` — возвращает количество сообщений.
- `getBalance()` — возвращает баланс контракта.
- `withdraw()` — выводит ETH владельцу контракта.

## Проверка

Минимальный набор тестов:

1. Развернуть контракт и проверить, что `owner` равен адресу деплойера.
2. Вызвать `writeMessage("Test")` и проверить увеличение количества сообщений.
3. Вызвать `writeMessage("")` и убедиться, что транзакция отклоняется.
4. Отправить сообщение с `value` и проверить рост баланса контракта.
5. Вызвать `withdraw()` владельцем и проверить обнуление баланса.
6. Попробовать вызвать `withdraw()` другим аккаунтом и убедиться, что доступ запрещен.
